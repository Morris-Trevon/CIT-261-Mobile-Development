Designing, Defining, and Triggering CSS3 Animations without Custom Libraries

This page demonstrates fluency in the above topics.

Javascrip Libraries were used to do animations to various web objects when CSS could not. A few years ago Animated Gifs, Adobe Flash and Javascript were the standard of doing animation and multimedia video and audio. However when HTML5 evolved and included multimedia support CSS3 also evolved to support animation and more. Adobe Flash was heavy on the browser and required another plugin while JavaScript was not for beginners required advanced knowledge, but pure CSS3 is run directly from the browser requiring no other plugin and easy to code for a novice. If your still curious about JavaScript libraries threejs.org has some cool animation using JavaScript, but this page will not demonstrate any of those. Let us begin!

Designing CSS3 Animations

Before designing any animation it should make sense and not mess up your content due to not making sense.

The following code in CSS will design the animation to apply to a HTML entity.

@keyframes bounce { // We first create the name
from {
transform: translateY(0px); // We now tell the animation from where to transform
}
to {
transform: translateY(-15px); // We then tell it where to stop
}
} // Transforming the object simply means changing the state
@-webkit-keyframes bounce { // Do the exact same thing but add -webkit- to ensure browser compatibility
from {
transform: translateY(0px);
}
to {
transform: translateY(-15px);
}
}

First we will place a common image with a simple animation.

.ballbounce { // This sets the properties of the ball
margin-top: 50px;
border-radius: 50%;
width: 300px;
height: auto;
background-color: transparent;
border: 0px solid #999;

}

.jump{ // Here we tell the class to use the animation called bounce and set the properties
animation-name: bounce; // specify the name [optional]
animation: bounce 1s infinite alternate; // you can set the name, time, how long, and alternate so it reverses the animation
-webkit-animation: bounce 1s infinite alternate;

}

As you can see a ball bouncing makes sense.

Triggering Animations

  What if we could make it not make sense by triggering a reaction?

@keyframes rotation { //New animation called rotation
50% {
transform: rotate(360deg); //New transform method called rotate
}
}

@-webkit-keyframes rotation {
50% {
transform: rotate(360deg);
}
}

We also need a class to load this into. I will explain why shortly.

.roll{ // The class called roll is not assigned to any html element
animation-name: rotation;
animation: rotation 5s infinite alternate;
-webkit-animation: rotation 5s infinite alternate;
}

// To get the button to work We have to use Javascript

function disruptAnimation(){
//Pull the HTML ID into JavaScript to be toggled
var balledit = document.getElementById("ballbounce");
//Use the classList.toggle function to pull from the CSS file the roll animation class
balledit.classList.toggle('roll');
}
Ok click the button, one click will change it, and the other will revert it as this is a toggle script. It toggles between first class and the changed class.
